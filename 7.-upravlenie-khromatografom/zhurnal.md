# Журнал

## Журнал событий

Диалоговое окно **«Журнал»** предназначено для сквозной автоматической регистрации, хронологического учета и структурирования всех программных и аппаратных событий, происходящих в процессе функционирования аналитического комплекса. Нажмите кнопку **«Журнал»** на верхней панели инструментов окна **«Управление хроматографом»** для вызова данного интерфейса.

Постоянное ведение журнала позволяет оператору контролировать действия персонала, отслеживать точное время смены технологических режимов и проводить ретроспективный анализ при возникновении спорных результатов измерений или скрытых неисправностей.

#### Структура таблицы событий

Каждая запись в системном журнале представляет собой монолитную строку данных, разделенную на следующие информационные колонки:

* **ID** — уникальный порядковый номер записи в текущей сессии работы.
* **Время** — точное системное время регистрации события ПК в формате `чч:мм:сс`.
* **Этап** — статус текущего технологического шага хроматографа в момент совершения события (например, _«Этап: Нулевой»_, _«Этап: Подготовка»_).
* **Время этапа** — относительный таймер обратного или прямого отсчета, фиксирующий, на какой именно секунде/минуте текущего этапа произошло событие.
* **Тип события** — графическая пиктограмма (иконка), наглядно классифицирующая природу сообщения (информационное, предупреждение, фатальный сбой, файловая операция).
* **Описание события** — детальный текстовый лог (например, _«Включение программы NetChrom v3.0»_, _«Хроматограф подключен»_, _«Открытие метода - 3816-ПИД-ПИД.metx»_).

#### Панель инструментов и фильтрация

Верхняя горизонтальная лента окна содержит элементы управления для быстрой навигации и очистки выводимого массива данных:

* **Кнопка «Обновить»** — принудительная актуализация списка событий в реальном времени.
* **Выбор даты** — раскрывающееся поле для выбора конкретного календарного дня. Позволяет оперативно загрузить архив событий за прошедшие рабочие смены.
* **Кнопка «Фильтр»** — активирует панель быстрой сортировки записей по типам (например, скрыть все информационные сообщения и оставить только ошибки).
* **Поле «Поиск по событиям»** — интерактивная строка контекстного поиска. Позволяет мгновенно отфильтровать таблицу по ключевому слову или коду ( например, при вводе слова _«методика»_ или _«испаритель»_ система скроет все остальные записи).
* **Настройки визуального выделения** — при нажатии на пиктограмму шестерни оператору открывается компактное меню индивидуальной настройки интерфейса, содержащее два ключевых чекбокса:
  * **Выделять этапы** — активирует автоматическое цветовое или графическое обособление строк, фиксирующих официальную смену технологических статусов прибора. Помогает визуально разделить хронологию на логические рабочие циклы.
  * **Выделять аварии и неисправности** — включает яркую предупреждающую подсветку для всех технологических параметров, вышедших за рамки установленных допусков (все сообщения со статусом _«вне нормы»_). Данная настройка гарантирует, что лаборант оперативно заметит факт температурного или газового сбоя в общем массиве логов.

#### Панель инструментов и фильтрация

В панели управления хроматографом добавляются элементы управления для работы с файлами отчетов:

* **Сохранить журнал в файл** — экспорт текущего лога событий в специализированный текстовый или табличный формат для последующей отправки в службу технической поддержки разработчика при решении сложных аппаратных проблем.
* **Открыть файл журнала** — запуск встроенного файлового менеджера для просмотра ранее сохраненных сторонних файлов логов без подключения к действующему хроматографу.
* **Печать журнала** — формирование унифицированной печатной формы отчета со сквозной хронологией событий текущей смены для подшивания в бумажный лабораторный журнал ведения анализов.
