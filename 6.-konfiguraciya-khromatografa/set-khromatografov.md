# Сеть хроматографов

## **Выбор и добавление хроматографа**

Модуль предназначен для централизованного мониторинга и добавления аналитического оборудования, подключенного к управляющей рабочей станции.

### Выбор и инициализация связи с хроматографом

При запуске программного комплекса «NetChrom 3.0» на экране отображается интерактивный перечень зарегистрированного и доступного в локальной сети оборудования. Для инициализации сессии обмена данными с конкретным хроматографом «Кристаллюкс-4000М» необходимо выполнить следующие действия:

1. **Идентификация устройства:** В центральной части окна найдите целевой хроматограф по его сетевому имени или заводскому серийному номеру (например, `3791_ПИД-ДТП`).
2. **Активация канала связи:** Напротив наименования прибора нажмите управляющую кнопку **«Управлять прибором»**.

После активации команды программный комплекс автоматически инициализирует фоновую сессию опроса телеметрии, откроет панель управления температурными режимами, расходами газов и модулями сбора данных.

### Управление пулом приборов (Конфигурация сети)

Для изменения состава лабораторного оборудования или оперативного переключения между приборами используется функциональный элемент **«Сеть хроматографов»**, расположенный в верхнем правом углу главного графического окна.

#### Доступные операции управления списком:

* **Развертывание сетевой структуры:** Нажатие на интерактивный маркер сети позволяет мгновенно свернуть или развернуть подробный древовидный список подключенных физических устройств и их измерительных каналов.
* **Добавление нового прибора:** Если целевой хроматограф отсутствует в перечне, нажмите кнопку **«Добавить хроматограф»**, расположенную на нижней навигационной панели, и укажите параметры его физического подключения (COM-порт).
* **Исключение из конфигурации:** Для удаления неиспользуемого или находящегося на техническом обслуживании оборудования выберите прибор в списке и нажмите кнопку **«Удалить из сети»**. Хроматограф будет исключен из общего контура оперативного мониторинга.

### Оперативный диспетчерский контроль

Встроенная система телеметрии выводит критически важные статусную информацию непосредственно в интерфейс сетевого модуля под уникальным именем каждого прибора:

* **Текущий технологический этап** (например, _«Этап: Охлаждение»_, _«Выход на режим»_, _«Готовность»_);
* **Временная метка** (общее время с момента инициализации текущего процесса или старта сессии анализа).

Данное архитектурное решение позволяет дежурному инженеру или начальнику лаборатории осуществлять сквозной визуальный контроль состояния всех аналитических комплексов предприятия в рамках единого рабочего окна.
