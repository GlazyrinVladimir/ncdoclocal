# Рабочий метод

## Рабочий метод и архитектура данных

В ПО «NetChrom 3.0» управление хроматографическим комплексом и обработка результатов построены на объектной концепции **Сквозного метода анализа**. Метод представляет собой структурированную совокупность аппаратных, расчетных и метрологических параметров, необходимых для проведения качественного и количественного анализа определенного типа пробы.

### Концепция разделения методов (Рабочий метод и Метод хроматограммы)

Для обеспечения гибкости аналитического цикла и сохранения прослеживаемости данных (требования стандартов GLP/GMP) в программном комплексе реализовано жесткое разделение на два типа методов:

1. **Рабочий метод (файл `*.metx`):** Хранится в специализированной директории базы данных или системной папке `Methods` конкретного хроматографа. На основе рабочего метода осуществляется автоматическое управление температурными режимами, расходами газов и подготовка прибора к анализу.
2. **Метод хроматограммы (внутри файла `*.chrx`):** При запуске анализа текущий рабочий метод со всеми его параметрами аппаратно копируется и «зашивается» непосредственно внутрь результирующего файла хроматограммы. Это гарантирует, что условия сбора данных и алгоритм обсчета навсегда останутся увязаны с конкретным аналитическим сигналом.

{% hint style="info" %}
При просмотре архивной хроматограммы оператор имеет право отредактировать параметры встроенного _метода хроматограммы_ (например, скорректировать калибровку или события интегрирования). При необходимости эти измененные параметры можно в один клик перенести обратно в глобальный _рабочий метод_ для использования в последующих анализах.
{% endhint %}

### Структура параметров, содержащихся в методе

Каждый файл метода (`*.metx` / `*.chrx`) включает в себя следующие обязательные информационные и конфигурационные блоки:

#### 1. Паспорт метода

* Идентификационное наименование метода, тип используемой модели хроматографа («Кристаллюкс-4000М»), автор разработки и развернутые инженерные комментарии к методике анализа.

#### 2. Аппаратные условия проведения анализа (Режим работы ГХ)

* Конфигурация активных детекторов и выходов АЦП.
* Программа температурного гидродинамического режима (многоступенчатое программирование термостата колонок).
* Параметры расходов и давлений газов-носителей, вспомогательных и поддувочных газов.
* Временные параметры продувки испарителей, переключения кранов-дозаторов и режимов работы автосамплеров (ДАЖ, ДАГ, ДРП).
* Заданная общая длительность регистрации хроматографического сигнала.

#### 3. Параметры интегрирования и обработки

* Критерии цифровой фильтрации шумов и компенсации дрейфа нулевой линии.
* Математические настройки автоматического поиска пиков (минимальная ширина, порог обнаружения, критерии разметки «плеч»).
* Временная таблица «событий интегрирования» для индивидуальной обработки сложных участков хроматограммы.

#### 4. Параметры качественной и количественной идентификации

* Выбранный тип математического расчета концентраций (абсолютная градуировка, внутренний стандарт, внутренняя нормализация).
* Справочник анализируемых компонентов (наименования, целевые окна поиска по абсолютному/относительному времени удерживания, индексы Ковача).
* Заданные функциональные группы компонентов и формулы расчета суммарных физико-химических свойств исследуемых сред.

#### &#x20;5. Настройки отчетности и валидации

* Конфигурация шаблонов итоговых паспортов анализов (состав табличных столбцов, выводимых данных по статистике и СКО).
* Параметры интерактивного вывода графических полей на экран и принтер.

{% hint style="warning" %}
Параметры рабочих методов жестко привязаны к физической конфигурации конкретного прибора. Попытка запустить рабочий метод, созданный для основного (базового) хроматографа, на удаленном приборе-сателлите (и наоборот) заблокирует инициализацию системы из-за аппаратного несовпадения конфигурации детекторов и плат расширения.
{% endhint %}
