# Создание метода

В программном обеспечении «NetChrom 3.0» процедуры создания, архивирования, верификации и хранения аналитических методик осуществляются через интегрированный **Файловый менеджер**. Модуль представляет собой централизованный интерфейс управления локальной или сетевой базой данных (СУБД), гарантирующий целостность и защиту конфигурационных файлов от несанкционированного изменения.

### Подготовка к созданию метода

Перед инициализацией процедуры создания нового рабочего метода (`*.metx`) оператор обязан выполнить предварительную проверку распределения аппаратных ресурсов:

1. **Аппаратная привязка:** Убедитесь, что в текущем рабочем пространстве программы выбран соответствующий **активный хроматограф** (или канал разделения).
2. **Мультиприборный контроль:** Инструментарий Файлового менеджера позволяет осуществлять сквозное переключение между доступными аналитическими комплексами в процессе работы. Помните, что геометрия колонок и состав детекторов выбранного прибора станут основой для валидации создаваемого метода.

### Алгоритм создания рабочего метода

Для инициализации и первичного конфигурирования нового файла метода выполните следующую последовательность действий:

1. **Вызов модуля:** На верхней панели инструментов главного окна (или в древовидной структуре объектов навигатора) выберите вкладку _«Файл»_ и активируйте команду **«Создать метод»** (также доступно через контекстное меню Файлового менеджера).
2. **Инициализация базы данных:** В открывшемся интерактивном окне Файлового менеджера выберите целевую директорию (папку `Methods` активного хроматографа) в структуре базы данных.
3. **Задание имени и спецификации:** В строке ввода укажите уникальное идентификационное наименование создаваемого метода, отражающее тип анализируемой пробы или нормативный документ (например, `ГОСТ_5542_Природный_газ`).
4. **Заполнение паспорта:** В появившейся экранной форме паспорта метода внесите обязательные метаданные: фамилию инженера-разработчика, тип хроматографа («Кристаллюкс-4000М») и краткое текстовое описание аналитической задачи.
5. **Фиксация объекта:** Нажмите кнопку **\[ОК]** (или клавишу **Enter**). Файловый менеджер автоматически сгенерирует новый защищенный объект базы данных с расширением `*.metx`, содержащий аппаратные параметры по умолчанию, и откроет его в режиме редактирования.

## Оперативное создание рабочего метода через модуль управления прибором

Дополнительно к стандартной процедуре через Файловый менеджер, программный комплекс «NetChrom 3.0» поддерживает сквозную инициализацию и создание новых рабочих методов непосредственно из интерактивной панели оперативного управления хроматографом. Данный метод позволяет оператору мгновенно переходить к изменению аппаратных параметров без закрытия текущей рабочей области мониторинга.

### Алгоритм действий в панели «Запуск анализа»

Для оперативного конфигурирования нового метода выполните следующую последовательность технологических шагов:

1. **Переход в модуль управления:** В главном окне управления подключенным хроматографом перейдите в функциональный раздел **«Запуск анализа»**.
2. **Локализация блока параметров:** В открывшейся панели найдите контекстный блок **«Параметры запуска»**, содержащий интерактивное поле выбора активной методики.
3. **Инициализация объекта:** Нажмите кнопку **\[Создать]** (кнопка снабжена пиктограммой нового документа). Программный комплекс автоматически сгенерирует пустой шаблон метода, установит на него фокус ввода и переведет интерфейс рабочей области в режим редактирования параметров созданного объекта.
4. **Аппаратная передача установок:** После завершения детального конфигурирования всех вкладок метода нажмите кнопку **\[Передать]** (кнопка снабжена пиктограммой стрелки/самолета телеметрии). Программа осуществит загрузку новых температурных и гидродинамических установок в электронные блоки хроматографа «Кристаллюкс-4000М», и прибор начнет автоматический цикл подготовки к анализу (выход на режим).

{% hint style="info" %}
Управляющая кнопка **\[Передать]** аппаратно блокируется системой и становится активной только после того, как в структуре метода будут заполнены все критические параметры (конфигурация детекторов, расходы газов) и файл будет успешно верифицирован и сохранен в базе данных Файлового менеджера.
{% endhint %}

## Многозадачность Файлового менеджера

Современное ядро Файлового менеджера ПО «NetChrom 3.0» поддерживает расширенную многозадачность. Данный функционал позволяет оператору одновременно открывать в едином рабочем пространстве несколько файлов хроматограмм, связанных с ними локальных методов (методов хроматограммы), а также независимых файлов глобальных рабочих методов (`*.metx`).

<div data-with-frame="true"><figure><img src="../.gitbook/assets/image (247).png" alt=""><figcaption><p>Несколько вкладок в файловом менеджере</p></figcaption></figure></div>

**Параллельное переключение и редактирование данных**

Все загруженные файлы распределяются по независимым горизонтальным вкладкам в верхней части экрана обработки. Программный комплекс обеспечивает сквозную синхронизацию процессов в режиме реального времени:

* **Оперативное переключение:** Оператор имеет возможность мгновенно переходить между вкладками различных хроматограмм для визуального сравнения графиков или параметров интеграции.
* **Редактирование в реальном времени:** Изменение настроек порогов чувствительности, добавление новых компонентов или корректировка временных ворот идентификации могут производиться непосредственно во вкладке открытого метода без остановки обсчета сопутствующих данных.
* **Цикл мгновенной переинтеграции:** При внесении изменений в параметры обработки оператору достаточно сохранить скорректированный метод и инициировать команду повторной обработки хроматограммы. Система выполнит автоматический пересчет площадей и концентраций по обновленным уставкам без необходимости перезапуска программного модуля.
