# Метод — Обработка

## Обработка

Вкладка **«Обработка»** предназначена для настройки алгоритмов обсчета хроматограмм. Здесь определяются правила, по которым программа выделяет пики из шума и рассчитывает их площадь и высоту.

В верхней части окна расположен переключатель (ползунок) **«Общие параметры для всех детекторов»**:

* **Включен:** настройки интегрирования и фильтрации будут применяться ко всем каналам (детекторам) в текущем методе одновременно. Это удобно для однотипных анализов.
* **Выключен:** позволяет задать индивидуальные параметры для каждого детектора (например, если ПИД требует высокой чувствительности, а ДТП — агрессивной фильтрации шума).

## Параметры интеграции

Подраздел: Параметры интегрирования

В этом блоке настраиваются критерии, по которым система распознает полезный сигнал и отделяет его от шумов и дрейфа базовой линии.

Основные параметры

* **Ширина пика на хроматограмме (сек) \[от ... до]:** Задает диапазон ожидаемой ширины пика на его полувысоте. Помогает алгоритму игнорировать слишком узкие (электрические наводки) или слишком широкие (размытые пятна или дрейф) сигналы.
* **Минимальная высота (мВ):** Порог отсечки по амплитуде. Пики, чья высота меньше указанного значения, не будут обрабатываться.
* **Минимальная площадь (мВ\*мин):** Порог отсечки по площади. Позволяет исключить из расчета мелкие примесные пики, которые не представляют интереса для анализа.
* **Задержка (мин):** Время от начала анализа, в течение которого интегрирование не производится. Обычно используется, чтобы исключить из расчета пик растворителя или переходные процессы в начале записи.

{% hint style="info" %}
**Совет по настройке:** Если программа «пропускает» нужные вам мелкие пики, попробуйте уменьшить значения в полях _Минимальная высота_ и _Минимальная площадь_, а также проверьте, не попадают ли они в интервал _Задержки_.
{% endhint %}

**Типы обработки (флаги)**

* **Отрицательные пики:** Включает инверсию алгоритма для регистрации пиков, направленных вниз от базовой линии (например, при использовании ДТП и газа-носителя с большей теплопроводностью, чем у компонента).
* **Базовая линия по долинам:** Метод прорисовки базовой линии, при котором она проводится по точкам минимума между неполностью разделенными пиками.
* **Аппроксимация по площади:** Метод расчета, при котором форма пика или его границы уточняются математически для более точного определения площади в сложных условиях.
* **Запрет наездников:** При включении этой опции программа перестает выделять малые пики («наездники»), находящиеся на склонах более крупных пиков, объединяя их площадь с основным.

В блоке **«Тип»** выберите необходимые параметры фильтрации и обработки сигнала:

* **Отрицательные пики** — включение режима детектирования пиков любой полярности (как выше, так и ниже базовой линии). Если флаг снят, программа будет игнорировать отрицательные отклонения, заменяя их расчетной базовой линией.
* **Базовая линия по долинам** — активация алгоритма, при котором базовая линия строится путем соединения локальных минимумов (впадин) между соседними пиками. Используется для разделения плохо разрешенных компонентов.
* **Аппроксимация по площади** — режим обработки зашкаленных пиков. Программа восстанавливает реальную форму пика и рассчитывает его площадь на основе анализа крутизны переднего и заднего фронтов.
* **Запрет наездников** — отключение автоматического поиска неразделенных пиков («наездников»). Весь массив данных между точками начала и конца детектирования будет рассматриваться как единый сложный пик.

**Наездник**

* **Поле ввода и переключатель (Вкл/Откл):** Позволяет задать количественный критерий (например, процентное отношение высот), по которому пик будет классифицироваться как «наездник». Если переключатель в положении **«Откл»**, разделение «наездников» производится по стандартному алгоритму.

## Фильтрация

Блок "фильтрация" предназначен для подавления аппаратурных шумов и очистки хроматограммы от случайных помех.

{% hint style="info" %}
Поскольку любая фильтрация искажает форму пиков, то ее использование оправдано только при высоких значениях шума.
{% endhint %}

**Важные условия применения:**

* **При анализе:** фильтрация применяется автоматически, если в диалоговом окне **Запуск метода** (вкладка «Обработка») активирован переключатель **Фильтрация**.
* **После анализа:** вы можете изменить параметры фильтрации или применить их повторно непосредственно к уже снятой хроматограмме в режиме обработки.

Описание доступных фильтров

В таблице можно комбинировать несколько методов, которые будут применяться последовательно (сверху вниз):

| Фильтр           | Описание и применение                                                                                                            |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| **Выбросов**     | Удаляет резкие «иглы» (одиночные скачки сигнала). Параметр «Ширина» здесь определяет порог чувствительности к амплитуде выброса. |
| **Медианы**      | Эффективен для подавления импульсного шума. В отличие от усреднения, лучше сохраняет крутизну фронтов пиков.                     |
| **Среднего**     | Классическое сглаживание. Усредняет значения в окне заданной ширины, снижая высокочастотные колебания базовой линии.             |
| **Полином**      | Сглаживание методом Савицкого-Голея. Минимизирует искажение формы и высоты пиков при эффективном подавлении шума.                |
| **Биномиальный** | Мягкое сглаживание по распределению Гаусса. Рекомендуется для работы с симметричными пиками на чистых хроматограммах.            |

Настройка параметров

Для выбора режима и глубины фильтрации выполните следующие действия:

1. <mark style="color:$danger;">**Активация:**</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">установите флаг в столбце</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">**«Вкл.»**</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">напротив одного или нескольких нужных фильтров.</mark>
2. **Установка ширины**.
   * Введите значение вручную или выберите его из выпадающего списка (если доступен).
   * **Ширина** определяет количество точек хроматограммы, которые подвергаются обработке за один шаг фильтра.
3. **Контроль результата:** при постобработке изменения отобразятся на графике сразу после применения настроек

{% hint style="info" %}
Чрезмерно высокая «Ширина» фильтров (особенно «Среднего» и «Биномиального») может привести к «заваливанию» пиков, уменьшению их расчетной высоты и потере разрешения между близко выходящими компонентами.
{% endhint %}

## События интеграции

Иногда подбором параметров интегрирования добиться корректной разметки пиков удается не на всей хроматограмме, а лишь на некотором ее участке (обычно в начале). В других случаях на хроматограмме могут присутствовать пики, не участвующие в расчете, а их разметка может мешать визуальному восприятию хроматограммы. Для тонкой настройки разметки пиков на хроматограмме могут быть использованы **События интегрирования**.&#x20;

Подраздел "События интегрирования" позволяет изменять логику обработки хроматограммы в режиме реального времени. Это необходимо, когда параметры, заданные в «Параметрах интегрирования», должны меняться на определенных участках анализа (например, из-за уширения пиков к концу хроматограммы или дрейфа базовой линии).

**Работа с таблицей**

* **Добавление события (`+`):** Создает новую строку в таблице.
* **Удаление события (`-`):** Удаляет выделенное событие.
* **Время начала / Время конца (мин):** Временной интервал, в течение которого действует выбранное правило.

{% hint style="info" %}
События выполняются в хронологическом порядке. Следите за тем, чтобы интервалы событий со схожими функциями не перекрывали друг друга без необходимости.
{% endhint %}

#### Описание типов событий

В колонке **«Тип события»** доступны следующие инструменты:

* **Ширина пика (сек.):** Позволяет изменить ожидаемую ширину пика для конкретного интервала времени. Актуально для длинных анализов, где поздние пики значительно шире первых.
* **Запрет разметки пиков:** Программа перестает искать и размечать любые пики в указанном интервале. Полезно для исключения зон выхода растворителя или «мусора» после обратной продувки.
* **Не убирать / Запрет обработки отрицательных пиков:** Управляет поведением системы при появлении сигнала ниже базовой линии.
* **Базовая линия по долинам:** Включает принудительное проведение базовой линии через минимумы между неразделенными пиками на данном участке.
* **Задержка обработки:** Приостанавливает работу интегратора.
* **Наездник:** Устанавливает критерии для определения малых пиков на плечах больших.
* **Минимальная высота / Площадь / Порог обнаружения:** Динамическое изменение чувствительности. Например, в конце анализа можно снизить порог, чтобы «вытянуть» очень малые компоненты.
* **Ширина плато базовой линии:** Настройка для стабильной фиксации базовой линии при наличии шумов или медленного дрейфа.
* **Режим одного пика:** Заставляет программу считать группу неразделенных пиков как один общий объект.

{% hint style="info" %}
Данный режим может быть полезен для объединения нескольких близко идущих пиков (например, изомеров, микропримесей и др.) в один. Для более сложных случаев можно воспользоваться объединением пиков в группы. В группы могут объединяться пики, стоящие в произвольном порядке, а не только подряд.
{% endhint %}

* **Установить базовую линию:** Принудительная фиксация положения базовой линии в конкретной точке времени.
* **Дрейф базовой линии / Запрет базовой линии:** Инструменты для компенсации наклонной или нестабильной нулевой линии.

**Поле «Значение»**

Для некоторых событий (например, _Ширина пика_ или _Минимальная высота_) необходимо ввести числовой показатель в эту колонку. Если событие является логическим (включить/выключить), поле может оставаться неактивным или принимать значения 1/0.
