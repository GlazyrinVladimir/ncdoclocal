# Общие настройки

Настоящий раздел руководства содержит описание параметров, определяющих глобальное поведение программного комплекса «NetChrom 3.0», политики автоматического именования файлов данных, правила разграничения прав доступа и базовые установки инициализации системы.

## Общесистемные настройки

Конфигурирование системных параметров выполняется во вкладке **«Общие»** главного диалогового окна настроек программы.&#x20;

### Состав конфигурируемых системных блоков

Параметры сгруппированы в четыре независимых интерактивных блока выпадающего типа:

* **Настройки интерфейса** — выбор глобальной языковой локализации программного обеспечения, конфигурация отображения всплывающих подсказок и адаптация графических панелей под различные разрешения мониторов.
* **Действия при анализе** — задание алгоритмов автоматизации при запуске и завершении сессии хроматографического разделения. Включает правила маскирования и автоматического присвоения имен файлам хроматограмм (генерация по дате, времени или номеру пробы).
* **Администрирование** — управление учетными записями пользователей и правами доступа в соответствии с требованиями стандартов целостности данных. Определение ролей для персонала лаборатории и защита методов от несанкционированного изменения.
* **Установки по умолчанию** — определение базовых шаблонов методов, конфигураций детекторов и градуировочных таблиц, которые автоматически загружаются в рабочее пространство при создании нового файла или инициализации программы.

## Настройки интерфейса

Блок управления визуализацией данных и единицами измерения.

* **Отобразить градуировочные коэффициенты** — включает вывод значений калибровочных факторов в таблицах и отчетах. Полезно для верификации расчетов концентраций.
* **Единица измерения давления** — выбор системных единиц (Атм, кПа, Бар и др.) для отображения давления в газовых магистралях прибора.
* **Ручное задание этапа Анализ** — при активации оператор получает возможность самостоятельно переключать фазы выполнения методики.
* **Выравнивать результаты до 100%** — дублирующий параметр глобальной настройки нормировки результатов (все найденные компоненты в сумме дают 100%).

## Действия при анализе

Группа параметров, автоматизирующих рутинные операции при выполнении серии измерений.

* **Автостарт метода после запуска** — автоматическая активация последнего использованного метода при включении ПО.
* **Автоувеличение номера пробы** — автоматическое инкрементирование (увеличение на единицу) числового индекса в названии пробы для каждого следующего анализа.
* **Запись хроматограмм в папки по дням** — активирует структурированное хранение данных: программа автоматически создает папки с текущей датой и сохраняет результаты анализа в них. Это упрощает поиск данных в архиве.
* **Резервирование** — включает режим создания страховых копий данных анализа в реальном времени или по завершении цикла.

## Администрирование

Параметры разграничения доступа и сервисного обслуживания.

* **Вход в программу с паролем** — активирует запрос авторизации при запуске ПО. Позволяет ограничить доступ посторонних лиц к настройкам и результатам анализов.
* **Сервисный режим** — специальный режим работы для инженеров и специалистов технической поддержки. Открывает доступ к расширенным настройкам аппаратной части хроматографа.

## Настройки по умолчанию

В данном блоке определяются стандартные метрологические интервалы и регламенты хранения временных данных.

### Параметры идентификации (окна допусков)

Эти значения используются программой для автоматического поиска пиков, если иные параметры не заданы в конкретной методике.

* **Окно времени (мин.)** — стандартная ширина временного интервала (в минутах) для идентификации компонента по абсолютному времени удерживания.
* **Окно отн. объема удерживания** — базовый допуск для поиска пиков при использовании метода относительных объемов.
* **Окно индекса** — значение допустимого отклонения для идентификации компонентов по индексам удерживания (например, индексам Ковача).

### Временные регламенты и лимиты

Параметры управления автоматизацией и жизненным циклом файлов.

* **Длительность запрета автозапуска методики (дней)** — устанавливает временной барьер для функции «Автостарт». Если с момента последнего анализа прошло больше указанного количества дней, автоматический запуск не произойдет. Это необходимо для обеспечения безопасности при длительных простоях оборудования (требуется ручная проверка готовности систем).
* **Хранить журнал хроматографа (дней)** — определяет срок хранения системных логов и записей о событиях прибора. По истечении этого срока записи удаляются автоматически.
* **Хранить резервированные файлы (дней)** — устанавливает период хранения страховых копий данных (результатов анализов и методов). Позволяет автоматически очищать диск от устаревших дубликатов, сохраняя только актуальный архив за последние дни.
