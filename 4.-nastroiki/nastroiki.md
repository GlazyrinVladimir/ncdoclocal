# Настройки

## Первоначальная настрйока программного комплекса

После завершения инсталляции и первого запуска ПО «NetChrom 3.0» необходимо выполнить базовое конфигурирование рабочей среды. Данная процедура выполняется однократно для синхронизации параметров программного обеспечения с аппаратной конфигурацией и составом модулей подключенного хроматографа (или пула приборов).

{% hint style="info" %}
Все параметры, заданные на этапе первоначальной настройки, являются динамическими. Они могут быть изменены или скорректированы уполномоченным пользователем на любом этапе последующей эксплуатации аналитического комплекса.
{% endhint %}

#### Инициализация модуля настроек

Для перехода к панели конфигурации параметров используйте кнопку **«Настройки»** (пиктограмма `⚙️` — «Шестерня»), расположенную в верхней панели управления текущего активного модуля прибора.

<div data-with-frame="true"><figure><img src="../.gitbook/assets/image (57).png" alt=""><figcaption><p>Символ настроек выглядит как "Шестерёнка"</p></figcaption></figure></div>

После нажатия кнопки **«Настройки»** программный комплекс автоматически откроет многовкладочное диалоговое окно параметров.

## Структура окна настроек

Многофункциональное диалоговое окно **«Настройки»** предназначено для глобального конфигурирования программной среды «NetChrom 3.0», оперативного управления параметрами автоматизации хроматографического анализа, разграничения прав доступа и контроля дискового пространства системы хранения данных.

### Архитектура интерфейса окна настроек

Рабочее пространство окна конфигурации разделено на две основные функциональные области:

1. **Левая панель (Навигатор):** Отображает иерархический список (древовидную структуру) доступных разделов и подразделов конфигурации программного комплекса.
2. **Правая панель (Рабочая область):** Динамически изменяемая контекстная вкладка, содержащая интерактивные параметры, таблицы и элементы управления, относящиеся к выбранному в левой панели пункту.

### Стартовый раздел «Главная»

При инициализации диалогового окна настроек верхнего уровня в правой панели по умолчанию активируется корневой раздел **«Главная»**.

Данный модуль является базовым и предназначен для выполнения следующих процедур:

* **Верификация лицензии:** Отображение текущего статуса лицензионного соглашения, регистрационного номера копии ПО и проверка аппаратного ключа защиты Guardant.
* **Системная логика:** Управление базовыми алгоритмами запуска приложения, инициализация фоновых потоков обработки данных и ведение сквозного системного журнала.
* **Контроль памяти:** Мониторинг заполнения локальной или сетевой базы данных хроматограмм, оценка доступного объема дискового пространства и вызов процедур архивации/очистки устаревших данных.

Подробный алгоритм действий при переполнении дискового пространства и правила переноса архивов хроматограмм приведены в разделе «Техническое обслуживание базы данных».

### Информационный блок лицензирования

В верхней части окна приведена системная информация о текущей версии ПО и параметрах лицензирования (Ключ лицензии, Организация-лицензиат). Данные сведения необходимы при обращении в службу технической поддержки.

* Строка **«NetChrom версия...»** — указывает текущий номер программной сборки и дату её релиза.
* Строка **«Ключ лицензии»** — отображает уникальный цифровой идентификатор установленной копии продукта.
* Строка **«Лицензирована для...»** — содержит официальное наименование предприятия-заказчика и его региональное расположение.
* Поле нормативной документации — информационная область, отображающая перечень поддерживаемых ГОСТов и методик измерений.

### Рекомендуемые параметры

Группа содержит настройки, определяющие логику выполнения анализа и способы математической обработки результатов. Использование данных параметров позволяет стандартизировать условия проведения измерений.

* **Автостарт метода после запуска**\
  Обеспечивает автоматическую загрузку и запуск последнего метода анализа, использованного в диалоговом окне «Запуск метода». Запуск происходит при старте программы или при повторном входе в неё после выхода.

{% hint style="info" %}
**Важное условие:** для корректной работы функции хроматограф должен быть включен.
{% endhint %}

* **Округлять температуры до сотых**\
  Специализированный параметр для хроматографов серии **К-4000М**. При активации точность отображения температур различных зон прибора в «Окне состояния» повышается до сотых долей градуса (0.01 °C).
* **Выравнивать результаты до 100%**\
  Режим автоматического математического выравнивания (нормировки) результатов анализа. При включении система приводит сумму всех определенных компонентов к 100%.
* **Контроль t° колонки и детектора**\
  Управляет системной проверкой соотношения температурных зон.
  * **Назначение:** Позволяет отключить стандартное условие безопасности.
  * **Применение:** Опция используется в случаях, когда методикой предусмотрен нагрев колонок до температур, превышающих температуру детектора.

### Хранилище

Обеспечивает мониторинг дискового пространства и настройку политик архивирования данных.

* **Индикатор заполнения** — отображает текущий объем базы данных (ГБ) и процент использования доступного пространства.
* **Автоматическая очистка** — позволяет установить регламентированный срок хранения хроматограмм и записей системного журнала. По истечении заданного интервала система производит безвозвратное удаление устаревших данных.
* **Очистка хранилища** — функция принудительного удаления временных файлов и оптимизации структуры базы данных в ручном режиме.

#### Управление конфигурацией

* **\[Применить]** — сохранение текущей конфигурации и вступление настроек в силу без перезагрузки приложения.
* **\[По умолчанию]** — восстановление "заводских" (базовых) установок параметров для текущего раздела.
