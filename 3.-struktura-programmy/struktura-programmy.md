# Структура программы

## Главное меню (старотовая страница)

Является центральным интерфейсным пространством приложения. Окно функционирует в двух режимах в зависимости от текущего состояния сессии:

* **Режим приветствия**: Отображается при первичном запуске программы (если в предыдущей сессии не было активных файлов). Содержит ссылки для быстрого перехода к созданию новых методов или открытию архивов.
* **Режим автовосстановления сессии**: Если ПО «NetChrom» было закрыто штатно, при следующем запуске режим приветствия автоматически пропускается. Программа инициализирует сквозное восстановление рабочих областей, открывая все хроматограммы, методы и таблицы, использовавшиеся в предыдущем сеансе.
* **Аналитическое пространство**: Служит единой подложкой, на которой разворачиваются графические поля хроматограмм, таблицы интеграции пиков и редакторы методов, вызванные из файлового менеджера.

## Файловый менеджер

Специализированное навигационное окно, предназначенное для каталогизации, поиска и управления внутренними типами файлов ПО «NetChrom» (файлы методов `*.metx`, файлы паспортов СО `*.grd`, архивные хроматограммы).

* **Панель фильтрации и поиска (Верхняя панель)**: Включает инструменты сортировки данных по временным интервалам, выпадающий список выбора конкретного метода, вкладки активных конфигураций и строку контекстного поиска.
* **Таблица результатов (Основная левая область)**: Отображает структурированный список доступных анализов с указанием метаданных: **«Дата и время»**, **«Проба»**, **«Объем»**, **«Оператор»**, **«Назначение»** и привязанный **«Метод»**.
* **Боковая панель статуса и дерева папок (Правая область)**:
  * Верхний блок визуализирует текущее аппаратное состояние выбранного хроматографа.
  * Нижний блок представляет собой интерактивное древо каталогов, разделенное на логические категории: _«Методы»_, _«Паспорта СО»_, _«Архив»_.
* **Логика взаимодействия**: Выбор и двойной клик на любом элементе в окне файлового менеджера инициализирует его открытие и визуализацию на пространстве **Главного окна программы**.

## Управление хроматографом

Окно предназначено для оперативного контроля текущего состояния аппаратной части хроматографического комплекса «Кристаллюкс-4000М», задания технологических режимов, ввода метаданных пробы и непосредственного запуска процедуры анализа. Окно имеет развитую блочную структуру.

Верхняя панель инструментов

Расположена в верхней части окна и содержит пиктограммы для быстрого вызова основных управляющих команд и сервисных утилит:

* **«Выполнить анализ»** — инициация одиночного измерения или запуск серии анализов.
* **«Подготовка/Выход на режим»** — перевод исполнительных узлов прибора в рабочее состояние.
* **«Неисправности»** — вызов диагностического журнала аппаратных ошибок.
* **«Журнал»** — открытие системного лог-файла событий хроматографа.
* **«Конфигурация хроматографа»** — переход к настройкам аппаратного состава.
* **«Стоп (Стоп/Авария)»** — экстренное прерывание текущего процесса или сброс термостатов.
* **«+5 минут к анализу»** — оперативное продление времени регистрации сигнала текущего измерения.
* **«Режим охлаждения»**, **«Продувка колонки»**, **«Режим сна»** — сервисные макрокоманды для управления температурным и газовым балансом прибора.
* **«Сеть хроматографов»** — выпадающий список для переключения между доступными в локальной сети приборами.

2\. Левая рабочая область (Параметры запуска и метаданные)

Область служит для конфигурирования условий текущего ввода и заполнения паспорта анализируемого образца.

* **Блок «Запуск анализа»**:
  * Переключатель _«Режим серии анализов»_ — активирует режим автоматического выполнения последовательности измерений.
  * Поле _«Метод»_ — выпадающий список для выбора управляющего файла, содержащего температурную и газовую программы. Под полем расположены кнопки _«Открыть»_, _«Создать»_, _«Изменить»_.
  * Поле _«Объем пробы»_ — ячейка для ввода фактического объема дозируемой пробы.
  * Выпадающий список _«Назначение хроматограммы»_ — определение типа выполняемой процедуры .
* **Блок «Паспорт пробы»**:
  * Набор текстовых и числовых полей для идентификации образца: **«Проба/Анализ»**, **«Номер пробы»**, **«Номер анализа»**, **«Дата отбора»**, **«Время отбора»**, **«Оператор»**, **«Место отбора пробы»**.
  * Текстовое поле **«Комментарий»** — ввод произвольных примечаний лаборанта к текущему анализу.

3\. Правая рабочая область (Мониторинг состояния в реальном времени)

Область предназначена для непрерывной визуализации физических параметров прибора. Разделена на две вкладки: _«Состояние»_ и _«Диагностика»_.

Вкладка **«Состояние»** включает следующие информационные панели:

* **Панель «Температуры»**: Таблица контроля нагревательных зон прибора. Для каждого узла в параллельных столбцах выводятся текущее измеренное значение и заданная методом уставка.
* **Панель «Расход газов»**: Информационная матрица блока ЭУПГ. Отображает расходы газов-носителей и вспомогательных газов в формате «Измерено / Задано».
* **Аппаратный виджет (Правый край)**:
  * Отображает имя прибора, его модель и активный рабочий метод.
  * Визуализирует состояние каналов детекторов с выводом текущего сигнала в милливольтах.
  * Содержит индикатор **«Параметры вне нормы»** — в штатном режиме выводится сообщение: `«Все параметры в норме»`. При отклонении температур или расходов индикатор меняет цвет и сигнализирует об ошибке выхода на режим.

## Настройки

Окно предназначено для конфигурирования глобальных параметров интерфейса ПО «NetChrom», управления правами лицензирования, администрирования встроенной базы данных (хранилища) и установки правил математического округления результатов обработки. Окно имеет двухпанельную структуру, состоящую из левого навигационного меню и главной рабочей области.

1\. Левое навигационное меню

Представляет собой вертикальный список вкладок для быстрого переключения между тематическими группами настроек приложения:

* **«Главная»** — базовая страница с информацией о системе, конфигурацией интерфейса и параметрами архивации.
* **«Хроматограмма»** — параметры визуального отображения графического поля (цвета линий, шрифты, правила масштабирования сеток).
* **«Формат таблиц»** — настройка разрядности числовых данных и состава столбцов в выходных аналитических матрицах.
* **«Общие»** — системные параметры программы (выбор языкового пакета, пути к временным папкам ОС).

2\. Главная рабочая область (Вкладка «Главная»)

Содержит три изолированных функциональных блока:

* **Блок «Лицензионная информация»**:
  * Отображает официальное наименование ПО и текущую версию сборки.
  * Содержит уникальный **«Ключ лицензии»** и сведения о владельце регистрационного удостоверения.
  * Информационная строка для вывода нормативной документации и поддерживаемых методик выполнения измерений.
* **Блок «Рекомендуемые параметры»**:\
  Набор триггерных переключателей (тумблеров) для управления логикой обработки данных программы:
  * _«Автостарт метода после запуска»_ — включает автоматическую инициализацию температурно-газового профиля при старте приложения.
  * _«Округлять температуры до сотых»_ — задает разрядность отображения физических параметров прибора в окнах мониторинга.
  * _«Выравнивать результаты до 100%»_ — активирует автоматическую нормировку суммарного содержания идентифицированных компонентов при расчете методом внутренней нормализации.
  * _«Контроль t° колонки и детектора»_ — включает программный аудит стабильности температурных зон.
* **Блок «Хранилище»**:\
  Предназначен для администрирования встроенной библиотеки методов и хроматограмм.
  * Индикатор дискового пространства — визуализирует объем занятой и свободной памяти на жестком диске в абсолютных (**ГБ**) и относительных (**%**) значениях.
  * Выпадающий список _«Удалять хроматограммы через»_ — настройка регламента автоматической очистки архива анализов.
  * Выпадающий список _«Очищать журнал прибора через»_ — определение жизненного цикла системного лог-файла ошибок.
  * Кнопка _«Очистить»_ — принудительное ручное освобождение дискового пространства хранилища.

3\. Нижняя панель управления

Содержит командные кнопки для сохранения и сброса внесенных изменений:

* **«По умолчанию»** — автоматический сброс всех текущих конфигураций программы к заводским системным настройкам разработчика.
* **«Применить»** — фиксация и запись измененных параметров в конфигурационный файл ПО без закрытия диалогового окна.

## Окно справочной системы

Справочная система представляет собой интегрированное электронное руководство пользователя, реализованное на платформе GitBook. Окно имеет современную трехпанельную структуру, обеспечивающую сквозную навигацию, контекстный поиск и интерактивное чтение документации.

1\. Верхняя панель навигации

Расположена в верхней части окна. Содержит логотип ПО (_«NetChrom v3.0»_) и центральную **Интерактивную строку поиска**.

* Поиск активируется кликом мыши или комбинацией горячих клавиш **\[Ctrl + K]**.
* Обеспечивает мгновенный полнотекстовый поиск по ключевым словам и фразам во всех разделах справки.

2\. Левая панель навигации (Иерархическое оглавление)

Содержит сквозное древовидное меню всех глав и подразделов технической документации. Позволяет пользователю переключаться между глобальными темами.

3\. Центральная информационная область

Основное текстовое поле, в котором разворачивается содержание выбранной статьи.&#x20;

4\. Правая панель навигации (Локальное содержание страницы)

Динамический блок _«ON THIS PAGE»_ (На этой странице). Отображает интерактивное оглавление текущей открытой статьи (по заголовкам уровней H2/H3). Позволяет пользователю в один клик перемещаться к нужной подразделу внутри длинной статьи.
