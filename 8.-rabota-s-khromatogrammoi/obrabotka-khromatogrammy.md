# Обработка хроматограммы

Данный раздел описывает последовательность действий оператора и внутреннюю логику функционирования программного комплекса «NetChrom» при первичной, автоматической и экспертной обработке зарегистрированных данных детектора.

## Автоматическая обработка по Рабочему методу

При первичном открытии сырого файла хроматограммы из архива базы данных через Файловый менеджер или сразу после автоматического завершения текущего физического измерения на приборе, программный комплекс «NetChrom» запускает регламентированный цикл автоматической интеграции:

1. **Считывание глобального шаблона:** Интегратор обращается к привязанному внешнему Рабочему методу (`*.metx`) и импортирует из него все базовые пороги чувствительности, фильтры шума и события.
2. **Поиск границ и расчет геометрии:** Математический алгоритм сканирует график, находит точки начала отклонения сигнала от нулевой линии (начала пиков), вершины и точки их возврата к фону (концы пиков). На основе этих координат автоматически рассчитывается ширина пиков и вычисляются их площади (`мВ*мин`) и высоты (`мВ`).
3. **Первичная разметка компонентов:** На основе рассчитанных площадей софт обращается к калибровочной таблице рабочего метода. Если в методе уже были заранее созданы компоненты и их справочные времена удерживания совпали с фактическим выходом пиков на графике, программа автоматически присваивает им имена и выводит предварительные концентрации.

## Итеративный алгоритм «обучения» метода

В реальной лабораторной практике идеальный Рабочий метод невозможно создать «вслепую» на пустом приборе. Его разработка всегда идет по циклической цепочке «обучения» системы на основе реальных хроматограмм:

1. **Запись хроматограммы:** Лаборант записывает анализ (или градуировочную смесь) по базовому, «пустому» рабочему методу, где настроены только температуры и газы. На выходе получается график с безымянными пиками.
2. **Обработка:** Оператор открывает этот файл. Программа изолирует настройки во внутренний **Метод хроматограммы**. Работая в этой безопасной зоне, лаборант подбирает идеальную чувствительность, прописывает события интегратора и вручную заносит правильные имена компонентов в таблицу по их временам удерживания.
3. **Фиксация эталона:** Когда хроматограмма стала идеально размеченной, оператор выполняет команду экспорта: _«Сохранить метод хроматограммы как Рабочий метод»_.
4. **Конвейерный автообсчет:** Внешний файл `*.metx` перезаписывается. Теперь система «обучена». Все последующие хроматограммы, которые прибор начнет записывать со следующей секунды, будут автоматически выходить сразу правильно размеченными и с подписанными веществами.

## Режимы обработки хроматограммы

Для оперативного управления результатами обсчета в верхней панели инструментов предусмотрена кнопка **«Обработать»**. Нажатие на кнопку открывает выпадающее контекстное меню выбора режимов пересчета данных:

* **Команда «Обработать»:** Запускает процедуру математического пересчета площадей и концентраций компонентов с использованием тех настроек, фильтров и порогов интегратора, которые оператор установил локально в текущем рабочем сеансе. Обсчет производится строго по внутреннему _**Методу хроматограммы**_. Данный режим предназначен для индивидуальной «тонкой» доводки результатов конкретного сложного измерения.
* **Команда «Обработать по рабочему методу»:** Инициализирует принудительный импорт параметров из внешнего эталонного файла методики (`*.metx`). При этом вся текущая локальная разметка открытого файла полностью замещается глобальными настройками интегратора. Режим является обязательным, если в лаборатории была обновлена общая градуировка прибора и оператору необходимо пересчитать серию ранее записанных архивных хроматограмм по новым, актуальным калибровочным коэффициентам.
* **Команда «Обработать как предыдущую»:**  Предназначена для выполнения последовательной автоматической интеграции файлов по математическому шаблону последнего завершенного обсчета. Если данная строка отображается серым цветом (деактивирована), это означает, что текущий тип количественного расчета или конфигурация выбранного метода не поддерживают функцию пакетной обработки данных.
