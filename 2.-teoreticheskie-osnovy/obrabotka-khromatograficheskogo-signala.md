# Обработка хроматографического сигнала

## Общие положения и концепция обработки

Математическая обработка хроматографического сигнала в ПО «NetChrom» представляет собой стандартизированный сквозной процесс преобразования первичных данных АЦП в метрологически аттестованный протокол результатов анализа.

Вся процедура обработки базируется на двух ключевых принципах:

* **Автоматизация цикла** — после подачи команды (или завершения записи хроматограммы) программа выполняет весь массив вычислений самостоятельно, минимизируя влияние субъективного фактора оператора.
* **Методический контроль** — логика работы алгоритмов на каждом этапе жестко управляется настройками текущего **Рабочего метода**.

Для понимания общей структуры ПО «NetChrom», весь конвейер обработки данных условно разделен на пять функциональных модулей, которые выполняются строго друг за другом:

1. **Модуль первичной фильтрации**\
   Отвечает за «очистку» зарегистрированного сигнала. Его главная задача — сгладить случайные электрические и аппаратные шумы АЦП, подготовив хроматограмму к математическому анализу без искажения формы реальных пиков.
2. **Модуль интегрирования (Разметки)**\
   Выполняет геометрический анализ очищенного сигнала. Модуль находит границы пиков (точки отклонения от фона) и математически точно рассчитывает их площади, высоты и времена удерживания.
3. **Модуль качественной идентификации**\
   Связывает геометрию с химией. На основе рассчитанных времен удерживания программа ищет совпадения в базе данных метода и присваивает безымянным пикам названия конкретных химических компонентов.
4. **Модуль градуировки (Калибровки)**\
   Определяет математическую зависимость между откликом прибора (площадью пика) и реальным содержанием вещества. На этом этапе на основе анализа стандартных образцов вычисляются поправочные коэффициенты или строятся градуировочные графики.
5. **Модуль количественного расчета**\
   Финальный вычислительный узел. Используя площади идентифицированных пиков рабочей пробы и градуировочные данные, модуль рассчитывает окончательные концентрации веществ и формирует отчетную документацию.

{% hint style="info" %}
Такой модульный подход позволяет оператору при необходимости гибко вмешиваться в процесс (например, корректировать параметры разметки или изменять градуировочные коэффициенты вручную), не нарушая общую целостность структуры данных хроматограммы.
{% endhint %}

## Факторы, определяющие успешность автоматической обработки

Эффективность работы всех последующих автоматических модулей программы напрямую зависит от качества исходного хроматографического сигнала. ПО «NetChrom» обрабатывает данные с минимальной погрешностью при соблюдении следующих условий:

* **Достаточное разрешение пиков**\
  Для корректного разделения пиков по умолчанию (без применения специальных программных допусков) расстояние между вершинами соседних пиков должно превышать их суммарную полуширину.
* **Оптимальное соотношение сигнал/шум**\
  Для уверенной работы модуля идентификации высота целевого пика должна как минимум в 3 раза превышать максимальную амплитуду высокочастотного шума нулевой линии. Для следовых компонентов требуется предварительная ручная настройка чувствительности.
* **Стабильность динамического диапазона**\
  Вершины хроматографических пиков не должны иметь плоских срезов (эффект «сплющивания» или ограничения сигнала). Подобное искажение свидетельствует о выходе сигнала за пределы шкалы АЦП или перегрузке детектора, что делает расчет площади математически неверным.

## Границы применимости автоматического расчета

Оператору необходимо учитывать, что автоматический конвейер обработки — это математическая модель, которая не может учесть все внештатные ситуации аналитического процесса. Автоматический расчет требует контроля и корректировки со стороны персонала в следующих случаях:

1. **Экстремальный дрейф нулевой линии** — вызванный, например, резким изменением расхода газа-носителя или перегревом колонки. Приводит к смещению расчетной базовой линии относительно реального фона.
2. **Появление ложных (артефактных) пиков** — обусловленный «призрачными» пиками из-за загрязнения испарителя или вымывания жидкой фазы сорбента. Программа идентифицирует их как целевые компоненты, если их время удерживания совпадет с табличным.
3. **Залповые выбросы сигнала** — единичные аппаратные скачки (наводки в электросети), которые алгоритм может ошибочно принять за узкие хроматографические пики с нулевым временем полуширины.
