# Элементы интерфейса и органы управления

Программный комплекс **«NetChrom 3.0»** использует стандартизированный графический интерфейс операционных систем семейств Microsoft Windows 10/11. Взаимодействие оператора с программой осуществляется через систему унифицированных диалоговых окон, контекстных панелей и интерактивных графических элементов.

## Принятые условные обозначения и аббревиатуры

В тексте настоящего Руководства и в интерфейсе ПО используются следующие маркеры внимания и сокращения:

{% hint style="info" %}
Данные сведения носят ознакомительный характер и помогают эффективнее организовать работу аналитического комплекса.
{% endhint %}

{% hint style="warning" %}
На эту информацию следует обратить особое внимание. Игнорирование указаний может привести к порче хроматографической колонки, потере данных анализа или метрологическим ошибкам.
{% endhint %}

#### Перечень используемых сокращений:

* **ПК** — Персональный компьютер (управляющая рабочая станция).
* **АЦП** — Аналого-цифровой преобразователь.
* **СКО** — Среднеквадратичное отклонение.
* **РРГ / РД / ИРГ** — Регулятор расхода газа / Регулятор давления / Измеритель расхода газа.
* **ДАГ / ДАЖ / ДРП** — Дозатор автоматический газовый / Дозатор автоматический жидкостный / Дозатор равновесного пара (автосамплеры).
* **ВУФ** — Вакуумная ультрафиолетовая лампа (детектор).
* **Типы хроматографических детекторов:**
  * **ПИД** — Пламенно-ионизационный детектор.
  * **ДТП** — Детектор по теплопроводности (катарометр).
  * **ЭЗД** — Электронозахватный детектор.
  * **ТИД** — Термоионный детектор.
  * **ПФД** — Пламенно-фотометрический детектор.
  * **ФИД** — Фотоионизационный детектор.

#### Механика работы с кнопками:

* **Стандартная кнопка:** Интерактивное поле с текстовым или графическим указанием назначения. Активируется одиночным щелчком левой кнопки мыши (ЛКМ). Текущая активная кнопка (фокус ввода) подсвечивается системным контуром.
* **Кнопка панели инструментов:** Компактный элемент с пиктограммой, дублирующий основные команды главного меню. При наведении курсора на кнопку без клика автоматически отображается **всплывающая подсказка** с описанием функции.

## Стандартные органы управления и ввода данных

Для детального конфигурирования параметров в диалоговых окнах программы используются следующие стандартные элементы интерфейса Windows 10/11:

#### Строка ввода (Текстовое поле)

* **Внешний вид:** Прямоугольное поле ввода, снабженное текстовой меткой (слева или сверху).
* **Действие:** Для ввода или изменения данных щелкните внутри поля ЛКМ — в поле появится мигающий текстовый курсор. Введите числовое или текстовое значение с клавиатуры. Для подтверждения ввода и фиксации данных обязательно нажмите клавишу **Enter**.

#### Переключатель (Чекбокс / Флаг)

* **Внешний вид:** Квадратное поле индикации со связанным текстом справа.
* **Действие:** Имеет два стабильных состояния: _Включено_ (отображается галочка внутри контура) и _Выключено_ (поле пустое). Изменение состояния происходит по одиночному клику ЛКМ. Используется для активации/деактивации конкретных функций (например, «Включить расчет по внутреннему стандарту»).

#### Выпадающий список (Комбобокс)

* **Внешний вид:** Текстовое поле, снабженное в правой части кнопкой с пиктограммой в виде перевернутого треугольника `▼`.
* **Действие:** Клик по кнопке `▼` разворачивает скрытый перечень доступных значений (например, выбор типа детектора: ПИД, ДТП, ЭЗД). Выбор целевой строки осуществляется кликом ЛКМ, после чего список автоматически сворачивается, а выбранное значение фиксируется в поле.

#### Иерархический список (Дерево объектов / Навигатор)

* **Внешний вид:** Структурированная панель, отображающая данные и файлы в виде вложенного дерева (узел -> подузел -> элемент).
* **Действие:** Узлы, имеющие вложенные элементы, снабжены маркером раскрытия `+` (или стрелкой `▶` в интерфейсах Windows 11). Клик по маркеру разворачивает вложенную структуру, а сам маркер меняется на `–` (или `▼`). Одиночный клик по конечному элементу дерева загружает его в активное рабочее окно программы.

## Интерактивная навигация и контекстные функции

#### Полоса прокрутки (Скроллбар)

* **Внешний вид:** Вертикальная или горизонтальная шина на краях окон, содержащая ползунок и две кнопки со стрелками. Автоматически появляется в окнах таблиц или графиков, если объем информации превышает видимую область экрана.
* **Действие:** Перемещение ползунка осуществляется путем его захвата (нажать и удерживать ЛКМ) с последующим перетаскиванием, либо точечными кликами по стрелкам на концах полосы для пошаговой прокрутки.

#### Ползунковый орган управления (Слайдер)

* **Внешний вид:** Градуированная шкала с подвижным указателем-стрелкой. Используется для плавной регулировки параметров (например, уровня фильтрации шума или масштаба осей).
* **Действие:** Ползунок перемещается методом перетаскивания (захват ЛКМ) или шаговыми кликами мыши слева/справа от указателя на шкале.

#### Контекстное (выпадающее) меню правой кнопки мыши

* **Внешний вид:** Всплывающий список команд, формируемый непосредственно в точке нахождения курсора.
* **Действие:** Вызывается нажатием **правой кнопки мыши (ПКМ)** в любой активной графической зоне (например, на поле хроматограммы или внутри таблицы). Содержит быстрые команды, привязанные к конкретной точке: _«Сбросить масштаб»_, _«Добавить событие интегрирования»_, _«Идентифицировать пик»_. Выбор команды осуществляется кликом ЛКМ.

## Фиксация и отмена операций (Кнопки подтверждения)

В нижней части  конфигурационного диалогового окна могут быть расположены стандартные кнопки фиксации параметров:

* **Кнопка \[ОК]:** Применяет все внесенные оператором изменения параметров, сохраняет их во временную память метода и закрывает текущее диалоговое окно. Дублируется нажатием клавиши **Enter** на клавиатуре (если фокус ввода не находится в текстовом поле).
* **Кнопка \[Отмена] (Cancel):** Полностью аннулирует все изменения, внесенные в текущем сеансе работы с окном, возвращает параметры к исходным значениям и закрывает окно. Полностью дублируется нажатием клавиши **Esc** на клавиатуре.
