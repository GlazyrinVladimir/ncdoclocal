# !!!Метод хроматограммы

## Что такое метод

**Метод** — это группа параметров, необходимых для анализа определенной пробы.&#x20;

Поскольку каждый тип пробы по разному ведет себя в газовом хроматографе (некоторым пробам требуется более высокая температура термостата, а другим необходимо пониженное давление газа или другой детектор) для каждого определенного типа анализа необходимо создать свой уникальный метод.

**Рабочий метод** позволяет осуществлять полное управление хроматографом и подключаемыми к нему периферийными устройствами.&#x20;

Рабочий метод <mark style="color:$danger;">хранится в папке "Методы" каждого хроматографа в</mark> виде файла с расширением \*.metx. С его помощью проводится анализ и снимается хроматограмма.&#x20;

{% hint style="info" %}
Для каждого подключенного прибора должен быть создан отдельный метод. Следует иметь в виду, что метод, созданный для основного прибора, не запустится на сателлите и наоборот.
{% endhint %}

Некоторые из параметров, сохраняемых в методе, определяют, как будет обработана проба при использовании метода. Параметры, содержащихся в методе:

* [паспорт метода](metod-khromatogrammy.md#pasport): тип хроматографа, имя, комментарии;
* условия проведения анализа (режим работы хроматографа): [рабочие детекторы](metod-khromatogrammy.md#detektory), [время анализа](metod-khromatogrammy.md#detektory), [температуры](metod-khromatogrammy.md#temperatury), [расходы газов](metod-khromatogrammy.md#raskhod-gaza), [параметры продувки](metod-khromatogrammy.md#produvka) и т.д.;
* [тип количественного расчета](metod-khromatogrammy.md#raschyoty);
* анализируемые [компоненты](metod-khromatogrammy.md#komponenty-i-gruppy) и их характеристики (наименование, время удержания, окно поиска,  канал детектирования, метод расчета площадей пиков, физико-химические свойства и т.д.);
* [группы](metod-khromatogrammy.md#komponenty-i-gruppy) компонентов и их характеристики;
* [параметры обработки](metod-khromatogrammy.md#obrabotka) хроматограмм: ширина пиков, порог обнаружения пиков, фильтрация шумов, настройка событий интегрирования;
* [параметры идентификации пиков](metod-khromatogrammy.md#obrabotka);
* [внешние устройства](metod-khromatogrammy.md#konfiguraciya-ustroistv) и параметры их работы.

Анализ данных и <mark style="color:$danger;">параметры отчета</mark> также сохраняются в методе при его создании. Эти параметры описывают, как интерпретировать хроматограмму, созданную для пробы, и <mark style="color:$danger;">какой тип отчета необходимо напечатать.</mark>

{% hint style="info" %}
При работе с методом условия проведения анализа из него передаются в хроматограф. **Метод записывается в хроматограммах**, снимаемых по текущему методу.
{% endhint %}

Хроматограмма сохраняется в виде файла с расширением \*.chrx. В тот же файл хроматограммы записывается и рабочий метод со всеми параметрами(температуры колонок, испарителей, детекторов, расходы газов, параметры обработки и др.). Этот метод носит название - **метод хроматограммы**.

{% hint style="info" %}
Просматривая хроматограмму, можно отредактировать параметры **метода хроматограммы**, а в последующем перенести изменения в **рабочий метод**, по которому данная хроматограмма была записана.
{% endhint %}

### Детекторы

Раздел **Детекторы** предназначен для выбора детекторов, участвующих в анализе, и настройки их ключевых рабочих параметров.

**Описание параметров**

* **Время анализа** — поле для задания общей длительности регистрации хроматографического сигнала (в минутах). Данное значение определяет период сбора данных со всех активных детекторов.
* **Рабочие детекторы** — список доступных детекторов хроматографа. Для включения детектора в текущую конфигурацию измерения необходимо установить флаг (чекбокс) напротив его наименования.
* **Ток, мА** — область настройки электрических параметров детекторов по теплопроводности. В поле ввода указывается значение тока питания моста (в миллиамперах) для выбранного детектора (например, **ДТП**).

{% hint style="info" %}
Используйте для активации только нужных детекторов в методе.

<mark style="color:$danger;">Если детектор не выбран в этом разделе, запись хроматограммы по данному каналу производиться не будет.</mark>
{% endhint %}

<p align="center"><img src=".gitbook/assets/image (32).png" alt=""></p>

#### Дополнительные параметры детектора ДТП&#x20;

При работе с ДТП необходимо  задать значение тока моста.

<figure><img src=".gitbook/assets/image (166).png" alt=""><figcaption></figcaption></figure>

{% hint style="info" %}
Величина тока моста зависит от температуры детектора, расхода газа - носителя и типа газа-носителя. Чем больше значение тока моста, тем выше чувствительность детектора. Однако это значение не должно превышать:

* **75 мА**, если газ-носитель - **гелий или водород**,&#x20;
* **40 мА** для **аргона**
* **30мА для азота**.&#x20;
{% endhint %}

{% hint style="danger" %}
**Большее значение тока ДТП может привести к выходу детектора из строя.**&#x20;
{% endhint %}

При достижении значения тока моста больше, чем указано в разделе блокировка конфигурации хроматографа, на этапе подготовка или готовность в окне управления хроматографом во вкладке состояние будет выведено соответствующее предупреждение и хроматограф перейдет на нулевой этап работы.&#x20;

<mark style="color:$danger;">Для более жесткого контроля задания тока моста в программе по умолчанию установлено ограничение значения тока ДТП в зависимости от выбранного в конфигурации хроматографа типа газа - носителя, а именно какое бы значение тока моста не указывалось, программа не передаст хроматографу больше максимально допустимого значения для данного газа-носителя.</mark>



{% hint style="warning" %}
При использовании азота наличие даже следов кислорода (воздуха) при высоком токе быстро разрушает вольфрамовую нить.

Никогда не включайте ток моста, если через детектор не идет **поток газа**. Без охлаждения потоком нити сгорают мгновенно.
{% endhint %}

<figure><img src=".gitbook/assets/image (165).png" alt=""><figcaption></figcaption></figure>

#### <mark style="color:$danger;">Дополнительный параметр детектора ЭЗД, ФИД и др.</mark>

<mark style="color:$danger;">При работе с ЭЗД указывается чувствительность.</mark>&#x20;

### Температуры

Раздел **Температуры** предназначен для программирования нагрева основных узлов прибора: термостата колонок, испарителей (инжекторов) и детекторов. Настройки этого раздела определяют скорость прохождения пробы через систему и форму получаемых пиков.

**Описание параметров**

* **Колонка, °C** — задание температуры термостата колонок. В случае программирования температуры здесь указывается начальное значение.
* **Детектор (1, 2, 3), °C** — поля для задания рабочих температур установленных детекторов.

{% hint style="info" %}
Примечание: Температура детектора должна исключать конденсацию компонентов пробы и фазы, обычно она на 20-30 °C выше температуры колонки.
{% endhint %}

* **Испаритель (1, 2, 3), °C** — задание температуры узлов ввода пробы. Устанавливается исходя из температур кипения компонентов анализируемой смеси.
* **Рабочие колонки** — блок управления потоками и регистрацией сигналов для конкретных колонок:
  * **Вкл.** — колонка задействована в анализе, поддерживается заданный расход газа-носителя.
  * **Откл.** — колонка деактивирована.

{% hint style="info" %}
При работе с устройством захолаживания термостата колонок (кондиционером) возможен ввод значения температуры термостата колонок равное 0 град. при условии, что в Конфигурации хроматографа включен **переключатель Кондиционер**.&#x20;

При работе с крио клапаном возможен ввод отрицательного значения температуры термостата колонок при условии, что в Конфигурации включен **переключатель Крио клапан**.&#x20;
{% endhint %}

#### Программирование температуры колонки

Этот блок предназначен для управления тепловым режимом термостата колонок в ходе анализа. Правильная настройка программы температуры позволяет ускорить выход «тяжелых» компонентов и улучшить разделение сложных смесей.

**Управление**

* **Переключатель «Вкл»:**
  * Если переключатель активен, прибор будет следовать заданной в таблице программе (менять температуру по шагам).
  * Если выключен — термостат будет поддерживать одну постоянную температуру, указанную в параметрах метода (изотерма).



{% hint style="info" %}
_<mark style="color:$primary;">Использование градиента</mark>_ <mark style="color:$primary;"></mark><mark style="color:$primary;">позволяет сократить время анализа и «сжать» пики тяжелых компонентов, делая их более четкими.</mark>
{% endhint %}

<figure><img src=".gitbook/assets/image (16).png" alt=""><figcaption><p>Рисунок 5.11 - программируемые температуры колонки</p></figcaption></figure>

* Управление ступенями
  * **Кнопка (`+`):** Добавляет новую ступень (линию) в программу. Это позволяет создавать сложные многоступенчатые графики нагрева.
  * **Кнопка (`-`):** Удаляет выбранную ступень из программы.

**Параметры таблицы программирования**

Каждая строка в таблице представляет собой этап (ступень) нагрева или выдержки.

* **Температура (°C):** Начальная температура текущего этапа. Для первой строки это стартовая температура анализа.
* **Конечная температура (°C):** Целевое значение, до которого термостат должен нагреться (или остыть) на данном этапе. Если начальная и конечная температуры совпадают, этот этап будет считаться изотермическим (выдержка).
* **Время (мин.):** Продолжительность этапа. Это время, в течение которого прибор удерживает температуру на заданном уровне (после достижения «Конечной температуры»).
* **Скорость подъема (°C/мин.):** Скорость, с которой прибор переходит от начальной к конечной температуре.
  * Чем выше скорость, тем быстрее нагревается термостат, но важно учитывать технические возможности нагревателя.
* **Рассчитанное время анализа (мин.)** — информационное поле, отображающее суммарную длительность всех этапов программы в минутах.

{% hint style="info" %}
В некоторых методиках необходимо понижение температуры термостата колонок во время анализа. <mark style="color:$danger;">Для этого необходимо указать отрицательную скорость программирования, т. е. со знаком минус. (</mark><mark style="color:$danger;">**надо или не надо я уже не знаю)**</mark>
{% endhint %}

<figure><img src=".gitbook/assets/image (179).png" alt=""><figcaption></figcaption></figure>

Типовые сценарии настройки:

1. **Линейный нагрев:** Задается начальная температура (например, 50°C), конечная (например, 250°C) и скорость (например, 10°C/мин).
2. **Нагрев с выдержкой:** Сначала задается этап с одинаковыми температурами на 2–3 минуты (чтобы зафиксировать легкие компоненты), а затем добавляется строка с ростом температуры.

{% hint style="info" %}
При настройке программы следите, чтобы «Конечная температура» в любой из строк не превышала максимально допустимую температуру для вашей колонки (указана в её паспорте или на бирке).
{% endhint %}

### Расход газа

{% hint style="warning" %}
Содержимое вкладки зависит от настроек в Конфигурации хроматографа.

Наименования каналов соответствуют именам, заданным в конфигурации прибора. В зависимости от настроек они могут отображаться как «Газ-1», «Сброс» и т.д.
{% endhint %}

Вкладка **«Расходы газов»** предназначена для управления потоками газовых сред внутри хроматографа. Параметры этого раздела определяют скорость движения пробы по колонке, режим ввода пробы в испарителе и подачу газов для работы детекторов.

#### Задание расходов газов для насадочных колонок

Для насадочных колонок управление газовыми потоками осуществляется путем прямого задания объемного расхода (мл/мин). В данном режиме система поддерживает установленные значения стабильными на протяжении всего анализа.

**Описание параметров**

* **Газ-1, 2, 3... (мл/мин)** — поля для задания расхода газа-носителя через соответствующие линии хроматографа. В зависимости от конфигурации прибора, это могут быть:
  * **Испаритель** — основной поток газа-носителя через насадочную колонку.
  * **Газ-поддув** — дополнительный поток на выходе из колонки, который компенсирует «мертвый объем» детектора и улучшает форму пика.
  * **Обдув** — поток для продувки оптических узлов.
* **Водород (мл/мин)** — расход горючего газа для детекторов пламенного типа (ПИД, ПФД).
* **Воздух (мл/мин)** — расход газа-окислителя для поддержания пламени в детекторе.

{% hint style="info" %}
**Стабильность потока:** Прибор переходит в режим готовности только после стабилизации всех заданных расходов.

**Герметичность:** Если фактический расход или давление не достигают заданных значений, необходимо проверить систему на наличие утечек (особенно в узле крепления колонки и септы).
{% endhint %}

#### Задание расходов газов для капиллярных колонок

Настройка газового режима для капиллярных колонок требует учета аппаратной конфигурации прибора, параметров колонки и настройки системы деления потока (сброса).&#x20;

**Подготовка и базовые настройки**

* **Газовая схема и регуляторы:** Должна быть правильно указана конфигурация электронных регуляторов давления (ЭРД) или расходов (ЭРР).
  * **Типы аппаратных модулей** — перед началом настройки параметров убедитесь, какой тип электронного регулятора давления (ЭРД) установлен в вашей системе:
    * **ЭРД7** — основной тип регулятора. Предназначен для поддержания заданного давления в линии **после себя** (на выходе). Используется для подачи газа в испарители и детекторы.
      * **ЭРД8** — специализированный модуль. Поддерживает давление в линии **до себя** (на входе). Обычно применяется в специальных схемах газовых потоков для сброса избыточного давления.
      * **Ручной регулятор давления** — используется в старых хроматографах.
* **Геометрия колонок:** Длина, внутренний диаметр и толщина фазы. Давление и расход жестко связаны через сопротивление капилляра; ошибка в диаметре даже в 0.05мм приведет к недостоверным результатам расчета.

{% hint style="info" %}
При замене капиллярной колонки на колонку с другим внутренним диаметром или длиной, необходимо обязательно обновить данные о геометрии колонки в методе, иначе расчет давления и расходов будет некорректным.
{% endhint %}

* **Тип газа-носителя:** Физические свойства газов (вязкость, сжимаемость) различаются. Необходимо выбрать актуальный газ: _Азот_, _Аргон,_ _Гелий_ или _Водород_.
  * **Вспомогательные газы:** Для работы пламенных детекторов (**ПИД, ТИД, ПФД**) установите требуемые расходы водорода и воздуха для поддержания пламени.

### Газовый калькулятор

Встроенный **«Газовый калькулятор»** предназначен для моделирования, расчета и автоматической оптимизации параметров газовых потоков в хроматографической системе. В основе расчета лежит математическое моделирование поведения идеального газа в капиллярных колонках с учетом сжимаемости носителя, температурных режимов и геометрии линий.

**Функциональные возможности калькулятора**

В интерфейсе калькулятора параметры **«Давление»** (атм) и **«Расход»** (мл/мин) являются взаимно зависимыми величинами.

* **Приоритет ввода:** Вы можете зафиксировать один из параметров. Например, если ввести целевой расход (оптимальный для текущего диаметра колонки и типа детектора), калькулятор автоматически рассчитает необходимое начальное давление на входе.
* <mark style="color:$danger;">**Линейная скорость:**</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">Параллельно вычисляется средняя линейная скорость газа-носителя (см/с), что позволяет оптимизировать разделение по уравнению Ван-Деемтера.</mark>

<figure><img src=".gitbook/assets/image (185).png" alt=""><figcaption></figcaption></figure>

* В поле **«Коэффициент деления»** введите требуемое соотношение (например, `1:2`).
* Программа, используя ранее вычисленный расход через колонку, автоматически рассчитает и установит абсолютное значение в поле **«Сброс»** (мл/мин).

**Программирование давления**

При реализации градиентного температурного режима вязкость газа-носителя динамически изменяется при нагреве. Если удерживать давление постоянным, расход через колонку во время анализа будет неизбежно падать.

Для стабилизации хроматографического процесса калькулятор предлагает два режима:

* **Постоянный расход (Constant Flow):** Калькулятор рассчитывает временную программу изменения давления. На основании температурной программы термостата колонок прибор будет плавно и синхронно повышать давление на входе, удерживая объемный расход через колонку на строго постоянном уровне.
* **Программируемое давление (Advanced Pressure):** Позволяет оператору вручную задать ступенчатый или линейный рост давления.

Для задания изменения давления в ходе анализа нажмите кнопку **«Давление»**. В появившемся диалоговом окне настройте следующие параметры:

* **Начальное давление** — давление на входе в колонку в начале анализа.
* **Время изобарического участка** — период времени, в течение которого давление остается постоянным перед началом изменения.
* **Скорость программирования** — скорость подъема или <mark style="color:$danger;">снижения давления.</mark>
  * _<mark style="color:$danger;">Примечание: Допускается</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">**обратное программирование давления**</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">(снижение потока) — для этого укажите скорость со знаком «минус».</mark>_
* **Конечное давление** — целевое значение, которое должно быть достигнуто.

Система автоматически отображает расчетное время изменения давления на основе введенных данных. Из этого же окна можно вызвать «Газовый калькулятор» для оперативной корректировки параметров.

{% hint style="warning" %}
После переноса рассчитанных данных в активный метод анализа убедитесь, что фактические показания датчиков давления на панели прибора стабилизировались и соответствуют расчетным.
{% endhint %}

### Продувка&#x20;

Вкладка **«Продувка»** предназначена для автоматизации процесса пост-аналитической очистки колонок от высококипящих компонентов (ВКК), которые не являются объектами качественного или количественного анализа.

**Функциональное назначение**

Режим обеспечивает принудительный перенос соединений с большими временами удерживания из испарителя в колонку и их ускоренный выход к детектору. Использование данной вкладки позволяет:

* Минимизировать время общего цикла анализа;
* Исключить появление пиков «памяти» на последующих хроматограммах;
* Продлить срок службы аналитических колонок.

**Состав и настройка параметров**

Для активации режима необходимо перевести переключатель **«Включить продувку»** в активное состояние и заполнить следующие поля:

{% hint style="warning" %}
Наименования каналов соответствуют именам, заданным в конфигурации прибора. В зависимости от настроек они могут отображаться как «Газ-1», «Сброс» и т.д.
{% endhint %}

1. **Группа «Газы, мл/мин»:**
   * **Газ-1...Газ-N:** Значения расходов газов-носителей через каналы ЭУПГ в фазе очистки. Для эффективной «продувки» рекомендуется устанавливать расходы выше рабочих, обеспечивая максимально допустимую линейную скорость для используемого типа колонки.
2. **Давление на колонке (для капиллярных колонок):**
   * Позволяет задать избыточное давление газа-носителя на входе в колонку. Увеличение давления в этом поле напрямую ускоряет движение ВКК по аналитическому тракту.
3. **Время продувки, мин:**
   * Длительность этапа очистки, которая отсчитывается с момента завершения основного времени анализа.
4. **Температура колонки, град:**
   * Температура термостата в процессе продувки. Для ускорения выхода ВКК температуру обычно устанавливают на уровне финальной точки температурной программы (но не выше предела термостабильности фазы).

{% hint style="info" %}
При настройке параметров продувки убедитесь, что суммарный поток газа через детектор не приведет к его перегрузке или погасанию пламени (в случае ПИД/ПФД).
{% endhint %}

**Алгоритм работы прибора**

* **Инициация:** После окончания регистрации хроматограммы прибор автоматически переключает расходы газов и температуру термостата на значения, указанные во вкладке «Продувка».
* **Исполнение:** В течение заданного времени происходит интенсивное «промывание» аналитического тракта.
* **Завершение:** По истечении времени продувки прибор переходит в режим подготовки к следующему анализу, возвращаясь к исходным уставкам газов и температур, заданным в основных вкладках метода.

### Экономия

{% hint style="warning" %}
Наименования каналов соответствуют именам, заданным в конфигурации прибора. В зависимости от настроек они могут отображаться как «Газ-1», «Сброс» и т.д.
{% endhint %}

Функция **«Экономия»** предназначена для рационального использования газа-носителя без ущерба для качества хроматографического разделения. Данный режим особо актуален при работе с капиллярными колонками в режиме деления потока.

**Преимущества использования**:

* **Снижение затрат:** Расход газа из баллона в режиме ожидания или во время долгого анализа сокращается.
* **Увеличение ресурса фильтров:** Газовые фильтры очистки (от влаги и кислорода) работают значительно дольше за счет уменьшения объема прокачиваемого через них газа.

Режим экономии активируется с помощью переключателей (ползунков) для каждого этапа работы хроматографа отдельно. При включении функции **«Вкл.»** становятся доступными поля для ввода конкретных значений расходов или давления.

**Доступные этапы для настройки**:

1. **Экономия газа на этапе «Подготовка»**
   * _Когда работает:_ В период, когда прибор только выходит на заданные температурные и газовые режимы после включения или смены метода.
   * _Зачем использовать:_ Если выход на режим занимает длительное время, можно установить минимально безопасный поток газа для защиты колонки от перегрева без лишнего расхода.
2. **Экономия газа на этапе «Готовность»**
   * _Когда работает:_ Прибор уже вышел на рабочий режим и ожидает команды оператора или автосамплера на начало анализа («Старт»).
   * _Зачем использовать:_ Позволяет не тратить газ впустую, пока прибор простаивает в ожидании ввода пробы. При нажатии кнопки «Старт», программа автоматически переключит расход на рабочий (указанный в основном методе).
3. **Экономия газа на этапе «Анализ»**
   * _Когда работает:_ Непосредственно во время записи хроматограммы.
   * _Зачем использовать:_ Обычно применяется в испарителях с делением потока (Split/Splitless). Например, через несколько минут после начала анализа (после того как проба попала в колонку) можно автоматически снизить «сброс» газа в атмосферу, не влияя на качество разделения.

При переводе ползунка в положение **«Вкл.»**, рядом с названием этапа появляется возможность задать время включения режима экономии, через которое будут установлены заданные выше значения **расхода (мл/мин)**.

{% hint style="info" %}
При установке параметров экономии всегда следите за тем, чтобы поток газа-носителя через колонку оставался достаточным для предотвращения диффузии воздуха внутрь системы, особенно при высоких температурах термостата.
{% endhint %}

### Сплит

Вкладка **«Сплит»** предназначена для настройки параметров режима **ввода без деления потока** (Splitless). Настройка этой вкладки позволяет задержать включение сброса газа из испарителя, обеспечивая максимальный перенос пробы в колонку.

{% hint style="warning" %}
Наименования каналов соответствуют именам, заданным в конфигурации прибора. В зависимости от настроек они могут отображаться как «Газ-1», «Сброс» и т.д.
{% endhint %}

**Порядок настройки**:

1. **Задайте расходы газов (Газ-1...Газ-N):**\
   Установите значение **0 мл/мин** (или иное необходимое минимальное значение) для каналов сброса. Эти значения будут поддерживаться прибором в течение времени задержки.
2. **Задержка включения рабочих газов, сек:**\
   Укажите время, через которое прибор должен вернуться к основным рабочим параметрам метода. В течение этого интервала расходы будут соответствовать значениям, указанным в пункте 1. В этот период вся проба из испарителя направляется в колонку.
3. **Переключатель «Включить сплит»:**\
   Переведите в активное состояние для автоматического запуска режима при каждом вводе пробы.

**Алгоритм функционирования:**

* **Начало анализа:** После нажатия кнопки «Старт» устанавливаются расходы газов, указанные в данной вкладке (режим ввода без деления потока).
* **Исполнение задержки:** В течение заданного времени («Задержка включения рабочих газов») проба переносится из испарителя в колонку при закрытом или ограниченном сбросе.
* **Переход в рабочий режим:** По окончании времени задержки прибор автоматически устанавливает расходы газов, которые были заданы как основные в настройках метода. Происходит удаление остатков пробы из испарителя и переход к штатному режиму разделения.

## Компоненты и группы

Этот раздел предназначен для составления списка целевых веществ и их объединения в группы для суммарного расчета.

### Добавление компонента

1\. Работа с компонентами

<mark style="color:$danger;">В левой части окна</mark> находится список индивидуальных веществ (компонентов).

* **Создать (`+`):** Добавляет новую строку в таблицу компонентов. После добавления необходимо задать имя вещества, время удерживания и параметры окна идентификации.
* **Удалить (`-`):** Безвозвратно удаляет выбранный компонент из метода.
* **Поиск:** Строка быстрого фильтра. Введите название вещества, чтобы мгновенно найти его в длинном списке (особенно полезно при анализе сложных смесей, например, бензинов).

2\. Работа с группами

<mark style="color:$danger;">Правая часть окна п</mark>озволяет объединять несколько компонентов в одну группу (например, «Сумма углеводородов», «Целевые примеси» или «Группа БТК»).

* **Создать (`+`):** Создает новую группу. В настройках группы можно указать, какие именно компоненты в неё входят. Расчет будет вестись как по каждому веществу отдельно, так и суммарно по всей группе.
* **Удалить (`-`):** Удаляет группу. При этом сами компоненты, входившие в неё, остаются в списке «Компоненты».
* **Поиск:** Позволяет быстро найти нужную группу по названию.



Основные операции

**Как добавить компонент в метод:**

1. Нажмите кнопку **«Создать»** под заголовком «Компоненты».
2. В появившейся строке введите **Название** (например, _Метан_).
3. Укажите **Ожидаемое время** (в минутах), которое программа будет использовать для поиска пика на хроматограмме.
4. Настройте **Окно идентификации** — временной диапазон вокруг ожидаемого времени (например, ±5%), в пределах которого пик будет считаться именно этим веществом.

<mark style="color:$danger;">**Как объединить компоненты в группу:**</mark>

1. <mark style="color:$danger;">Нажмите</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">**«Создать»**</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">под заголовком «Группы» и задайте имя группы.</mark>
2. <mark style="color:$danger;">В настройках группы отметьте галочками нужные вещества из общего списка компонентов.</mark>
3. <mark style="color:$danger;">Результаты по этой группе (суммарная площадь, концентрация) будут автоматически отображаться в финальном отчете.</mark>

{% hint style="info" %}
Используйте поле **«Поиск»**, если в вашем методе более 20-30 компонентов. Это значительно ускоряет навигацию при корректировке времен удерживания.

### Параметры

Подраздел **«Параметры»** является основным справочником метода, где для каждого анализируемого вещества жестко задаются критерии его узнавания и принадлежность к аппаратным каналам.

Для каждого компонента заполняются следующие обязательные поля:

* **Имя компонента:** Уникальное текстовое наименование вещества, которое будет использоваться во всех расчетных таблицах и итоговом протоколе.
* **Тип компонента:** Определяет роль пика в математической модели метода.
  * Внутренний стандарт - компонент для расчета по методу внутреннего стандарта или компонент, относительно которого рассчитываются коэффициенты чувствительности по методу внутренней нормализации;&#x20;
  * Реперный - компонент или компоненты, относительно которых рассчитываются относительные времена, объемы удерживания или индексы;&#x20;
  * Основной - компонент, количество которого рассчитывается как разница между количеством пробы и суммарным количеством остальных компонентов или же используется как дополнительная метка компонента при некоторых видах дополнительных расчетов;&#x20;
  * Общий - условный компонент, к которому будут отнесены все неидентифицированные пики хроматограммы&#x20;
  * Свидетель - компонент, относительно которого корректируется объем введенной пробы, как правило это компонент с большим содержанием в пробе, например, этиловый спирт в водке;&#x20;
  * Запрет идентификации - данный пик не будет идентифицироваться на хроматограмме, чтобы устранить его из расчетов (вместо удаления пика или компонента);&#x20;
  * Специальный - компонент, концентрация которого выводится отдельной строкой в панели таблиц главного окна во вкладке Расчет,&#x20;
  * Подтверждающий компонент применяется для идентификации на двух каналах детекторов. Обычным (опорным) компонентом, принято выбирать на том канале, где пик компонента лучше отделяется от других.
* **Время удерживания:** Опорное значение времени в минутах, соответствующее максимуму концентрации компонента на выходе из колонки.
* **Окно времени (Временной допуск):** Интервал, определяющий границы поиска пика относительно заданного времени удерживания.
* **Рабочий детектор (каналы):** Указание конкретного детектора (ПИД, ДТП и др.), на котором будет производиться поиск и расчет данного компонента.
* **Группа (принадлежность):** Поле выбора, позволяющее на этапе создания компонента сразу включить его в состав определенной расчетной группы. Это обеспечивает автоматическое суммирование концентрации компонента в соответствующую строку отчета (например, отнесение «Бензола» к группе «Ароматика»).

## Обработка

### Параметры интеграции

Подраздел: Параметры интегрирования

В этом блоке настраиваются критерии, по которым система распознает полезный сигнал и отделяет его от шумов и дрейфа базовой линии.

Основные параметры

* **Ширина пика на хроматограмме (сек) \[от ... до]:** Задает диапазон ожидаемой ширины пика на его полувысоте. Помогает алгоритму игнорировать слишком узкие (электрические наводки) или слишком широкие (размытые пятна или дрейф) сигналы.
* **Минимальная высота (мВ):** Порог отсечки по амплитуде. Пики, чья высота меньше указанного значения, не будут обрабатываться.
* **Минимальная площадь (мВ\*мин):** Порог отсечки по площади. Позволяет исключить из расчета мелкие примесные пики, которые не представляют интереса для анализа.
* **Задержка (мин):** Время от начала анализа, в течение которого интегрирование не производится. Обычно используется, чтобы исключить из расчета пик растворителя или переходные процессы в начале записи.

{% hint style="info" %}
**Совет по настройке:** Если программа «пропускает» нужные вам мелкие пики, попробуйте уменьшить значения в полях _Минимальная высота_ и _Минимальная площадь_, а также проверьте, не попадают ли они в интервал _Задержки_.
{% endhint %}

**Типы обработки (флаги)**

* **Отрицательные пики:** Включает инверсию алгоритма для регистрации пиков, направленных вниз от базовой линии (например, при использовании ДТП и газа-носителя с большей теплопроводностью, чем у компонента).
* **Базовая линия по долинам:** Метод прорисовки базовой линии, при котором она проводится по точкам минимума между неполностью разделенными пиками.
* **Аппроксимация по площади:** Метод расчета, при котором форма пика или его границы уточняются математически для более точного определения площади в сложных условиях.
* **Запрет наездников:** При включении этой опции программа перестает выделять малые пики («наездники»), находящиеся на склонах более крупных пиков, объединяя их площадь с основным.

В блоке **«Тип»** выберите необходимые параметры фильтрации и обработки сигнала:

* **Отрицательные пики** — включение режима детектирования пиков любой полярности (как выше, так и ниже базовой линии). Если флаг снят, программа будет игнорировать отрицательные отклонения, заменяя их расчетной базовой линией.
* **Базовая линия по долинам** — активация алгоритма, при котором базовая линия строится путем соединения локальных минимумов (впадин) между соседними пиками. Используется для разделения плохо разрешенных компонентов.
* **Аппроксимация по площади** — режим обработки зашкаленных пиков. Программа восстанавливает реальную форму пика и рассчитывает его площадь на основе анализа крутизны переднего и заднего фронтов.
* **Запрет наездников** — отключение автоматического поиска неразделенных пиков («наездников»). Весь массив данных между точками начала и конца детектирования будет рассматриваться как единый сложный пик.

**Наездник**

* **Поле ввода и переключатель (Вкл/Откл):** Позволяет задать количественный критерий (например, процентное отношение высот), по которому пик будет классифицироваться как «наездник». Если переключатель в положении **«Откл»**, разделение «наездников» производится по стандартному алгоритму.

| **Выбросов**     | Удаляет резкие «иглы» (одиночные скачки сигнала). Параметр «Ширина» здесь определяет порог чувствительности к амплитуде выброса. |
| **Медианы**      | Эффективен для подавления импульсного шума. В отличие от усреднения, лучше сохраняет крутизну фронтов пиков.                     |
| **Среднего**     | Классическое сглаживание. Усредняет значения в окне заданной ширины, снижая высокочастотные колебания базовой линии.             |
| **Полином**      | Сглаживание методом Савицкого-Голея. Минимизирует искажение формы и высоты пиков при эффективном подавлении шума.                |
| **Биномиальный** | Мягкое сглаживание по распределению Гаусса. Рекомендуется для работы с симметричными пиками на чистых хроматограммах.            |

Настройка параметров

Для выбора режима и глубины фильтрации выполните следующие действия:

1. <mark style="color:$danger;">**Активация:**</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">установите флаг в столбце</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">**«Вкл.»**</mark> <mark style="color:$danger;"></mark><mark style="color:$danger;">напротив одного или нескольких нужных фильтров.</mark>
2. **Установка ширины**.
   * Введите значение вручную или выберите его из выпадающего списка (если доступен).
   * **Ширина** определяет количество точек хроматограммы, которые подвергаются обработке за один шаг фильтра.
3. **Контроль результата:** при постобработке изменения отобразятся на графике сразу после применения настроек

{% hint style="info" %}
Чрезмерно высокая «Ширина» фильтров (особенно «Среднего» и «Биномиального») может привести к «заваливанию» пиков, уменьшению их расчетной высоты и потере разрешения между близко выходящими компонентами.
{% endhint %}

### События интеграции

Иногда подбором параметров интегрирования добиться корректной разметки пиков удается не на всей хроматограмме, а лишь на некотором ее участке (обычно в начале). В других случаях на хроматограмме могут присутствовать пики, не участвующие в расчете, а их разметка может мешать визуальному восприятию хроматограммы. Для тонкой настройки разметки пиков на хроматограмме могут быть использованы **События интегрирования**.&#x20;

Подраздел "События интегрирования" позволяет изменять логику обработки хроматограммы в режиме реального времени. Это необходимо, когда параметры, заданные в «Параметрах интегрирования», должны меняться на определенных участках анализа (например, из-за уширения пиков к концу хроматограммы или дрейфа базовой линии).

**Работа с таблицей**

* **Добавление события (`+`):** Создает новую строку в таблице.
* **Удаление события (`-`):** Удаляет выделенное событие.
* **Время начала / Время конца (мин):** Временной интервал, в течение которого действует выбранное правило.

{% hint style="info" %}
События выполняются в хронологическом порядке. Следите за тем, чтобы интервалы событий со схожими функциями не перекрывали друг друга без необходимости.
{% endhint %}

#### Описание типов событий

В колонке **«Тип события»** доступны следующие инструменты:

* **Ширина пика (сек.):** Позволяет изменить ожидаемую ширину пика для конкретного интервала времени. Актуально для длинных анализов, где поздние пики значительно шире первых.
* **Запрет разметки пиков:** Программа перестает искать и размечать любые пики в указанном интервале. Полезно для исключения зон выхода растворителя или «мусора» после обратной продувки.
* **Не убирать / Запрет обработки отрицательных пиков:** Управляет поведением системы при появлении сигнала ниже базовой линии.
* **Базовая линия по долинам:** Включает принудительное проведение базовой линии через минимумы между неразделенными пиками на данном участке.
* **Задержка обработки:** Приостанавливает работу интегратора.
* **Наездник:** Устанавливает критерии для определения малых пиков на плечах больших.
* **Минимальная высота / Площадь / Порог обнаружения:** Динамическое изменение чувствительности. Например, в конце анализа можно снизить порог, чтобы «вытянуть» очень малые компоненты.
* **Ширина плато базовой линии:** Настройка для стабильной фиксации базовой линии при наличии шумов или медленного дрейфа.
* **Режим одного пика:** Заставляет программу считать группу неразделенных пиков как один общий объект.

{% hint style="info" %}
Данный режим может быть полезен для объединения нескольких близко идущих пиков (например, изомеров, микропримесей и др.) в один. Для более сложных случаев можно воспользоваться объединением пиков в группы. В группы могут объединяться пики, стоящие в произвольном порядке, а не только подряд.
{% endhint %}

* **Установить базовую линию:** Принудительная фиксация положения базовой линии в конкретной точке времени.
* **Дрейф базовой линии / Запрет базовой линии:** Инструменты для компенсации наклонной или нестабильной нулевой линии.

**Поле «Значение»**

Для некоторых событий (например, _Ширина пика_ или _Минимальная высота_) необходимо ввести числовой показатель в эту колонку. Если событие является логическим (включить/выключить), поле может оставаться неактивным или принимать значения 1/0.

### Дозатор автоматический газовый (ДАГ)

Вкладка **«ДАГ»** предназначена для настройки параметров работы автоматических дозаторов газовых проб. Настройки данного раздела определяют алгоритм забора пробы, объем и количество циклов анализа.

**Параметры выбора устройства**

* **Тип устройства** (верхний выпадающий список): Выбор модификации установленного дозатора (**ДАГ-3, ДАГ-4, ДАГ-10/12**). От выбора типа зависят доступные алгоритмы управления.
* **Активные модули** (второй выпадающий список): Позволяет выбрать конкретный ДАГ из списка доступных в системе («Первый», «Второй», «Третий»), если в конфигурации прибора задействовано несколько дозаторов.

{% hint style="info" %}
При работе сателлитом и наличии двух ДАГ, если для первого хроматографа выбирается ДАГ номер 1, то на втором хроматографе (сателлите) - номер 2.
{% endhint %}

**Режимы запуска и повтора**

* **Повторять непрерывно**: При включении данного режима (переключатель в положении «Вкл.») система будет автоматически инициировать новый цикл анализа сразу после завершения предыдущего. Используется для постоянного мониторинга потока.
* **Ручной старт серии анализов**: Активирует возможность ручного запуска последовательности анализов оператором.

Группа «**Параметры пробы**»

В данной группе задаются количественные характеристики цикла дозирования:

* **Объем пробы (мл.)**: Объем газовой пробы, который будет отобран дозатором для ввода в колонку.
  * Для точного определения этого значения воспользуйтесь функцией **«Калькулятор объема газовой пробы»** (вызывается нажатием кнопки рядом с полем ввода). Калькулятор позволяет рассчитать объем на основании физических параметров петли и условий отбора.
* **Число анализов**: Количество повторных циклов ввода пробы из одной серии.
  * Для непрерывных анализов включите переключатель Повторять непрерывно. Тогда число анализов будет неограниченно, а строка с числом анализов - недоступна.
* **Время отбора (сек.)**: Интервал времени, в течение которого происходит заполнение дозирующей петли или емкости пробой.

**Время переключения** (сек.)

Здесь настраиваются временные задержки для исполнительных механизмов конкретных модулей (**ДАГ-1, ДАГ-2, ДАГ-3**).

* Параметр определяет время, необходимое для физического перемещения привода дозатора в рабочее положение и обратно.
* **Примечание:** Значения устанавливаются в соответствии с техническим паспортом на конкретную модель дозатора (обычно настраиваются один раз при пусконаладочных работах).

#### Порядок настройки ДАГ

Для подготовки автоматического дозатора к работе выполните следующие действия:

1. **Выбор модели**: В верхнем раскрывающемся списке выберите тип установленного дозатора (например, **ДАГ-3**).
2. **Выбор активного модуля**: Во втором списке выберите порядковый номер дозатора в системе (**Первый**, **Второй** и т.д.).
3. **Выбор режима запуска**:&#x20;

* Активируйте **«Ручной старт серии анализов»**, если планируете запускать цикл по команде оператора.
* Активируйте **«Повторять непрерывно»**, если требуется автоматический циклический анализ потока.

4. **Расчет и ввод объема**:

* При необходимости нажмите на иконку **Калькулятора** рядом с полем «Объем пробы».
* Введите полученное значение в поле **Объем пробы (мл.)**.

5. **Задание цикла**: Укажите **Число анализов** и **Время отбора**.
6. **Настройка временных задержек**: В блоке **«Время переключения»** установите время срабатывания приводов (в соответствии с паспортом прибора).

### Кран обратной продувки/переключающий кран

Вкладка **«Краны»** предназначена для управления пневматическими или электрическими кранами-переключателями и кранами обратной продувки, установленными на хроматографе. Настройки данной вкладки определяют состояние кранов в момент запуска метода и их временные характеристики.

В зависимости от аппаратной конфигурации, краны разделены на группы по их функциональному назначению.

#### Краны обратной продувки

Используются для изменения направления потока газа-носителя в колонке (предколонке) для сброса тяжелых компонентов или защиты аналитической колонки.

* **Кран (№)**: Порядковый номер физического подключения крана к контроллеру.
* **Время переключения**: время переключения крана на обратную продувку.
  * _Примечание:_ Справа от поля ввода из выпадающего списка можно выбрать единицы измерения (**сек.** или **мин.**).
* **Состояние (Вкл./Откл.)**: Переключатель активации крана в текущем методе.
  * **Вкл**: Кран задействован в методе. Программное обеспечение будет подавать сигналы управления на данный кран согласно временной программе.
  * **Откл**: Кран деактивирован. Управление данным механизмом в рамках текущего метода не осуществляется.

#### Переключающие краны

_Примечание: Данный тип устройств настраивается аналогично кранам обратной продувки, но имеет расширенные временные параметры._

Для переключающих кранов в интерфейсе доступны **два параметра времени переключения**:

* **Время переключения 1**: Время перехода из исходного состояния в рабочее.
* **Время переключения 2**: Время возврата крана в исходное состояние.

Эти параметры позволяют более точно настроить работу механизмов с разной скоростью срабатывания или специфическими требованиями к циклу дозирования.

**Порядок настройки**

1. Установите исходное состояние для каждого крана, задействованного в методе.
2. Введите время переключения в соответствии с техническими данными установленных приводов.
3. **Важно:** Убедитесь, что выбранные единицы измерения (минуты или секунды) соответствуют вашему алгоритму анализа.
